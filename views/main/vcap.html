VMware's Certificate Application Platform

VCAP (VMware's Cloud Application Platform) adalah sebuah ikatan layanan dan juga kernel dari Cloud Foundry sebagai komponen pembangun sebuah PaaS dan pendukung fungsionalitasnya.

Di dalamnya terdapat bagian-bagian seperti Cloud Controller, Router, DEA, Health Manager, Service Gateway, Service Node dan lain-lain. Dan kesemuanya saling bekerja sama untuk membentuk sistem PaaS.

Cloud Controller

Cloud Controller (CC) dapat dikatakan sebagai unit otak dari kernel.

Router

Router adalah sebuah unit yang juga berperan sebagai pengatur rute.Router menggunakan Nginx sebagai load balancer. Router tersedia di git hub https://github.com/cloudfoundry/router

DEA(Droplet Execution Agent)

DEA (Droplet Execution Agent) sebagai unit yang mengolah permintaan dan perintah dari CC dalam bentuk droplet , droplet akan diproses dan diolah menjadi aplikasi. DEA tersedia di git hub https://github.com/cloudfoundry/dea

Health Manager

Health Manager adalah unit yang mengawasi kesehatan dari unit DEA yang mengurusi aplikasi.

Service Gateway

Service Gateway adalah unit yang menyediakan saluran komunikasi antara CC dengan layanan-layanan dalam PaaS yang sudah tersedia. Service Gateway tersedia di git https://github.com/cloudfoundry/vcap-services

Service Node

Service Node adalah unit yang terdapat di belakang Service Gateway. Service Node juga tersedia dalam git https://github.com/cloudfoundry/vcap-services

Client

VCAP dapat diakses dalam melalui beberapa client seperti VMC (Command Line Console), Eclipse/STS Plugin, Web Interface.
